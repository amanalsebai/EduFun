# 01 — المعمارية وتدفّق البيانات

## 1. لماذا ننتقل من تخزين محلي إلى باك إند؟

التطبيق حالياً يحفظ كل شيء في `SharedPreferences` (تخزين محلي مفتاح/قيمة):

```dart
// من untitled1/lib/core/utils/score_manager.dart
final prefs = await SharedPreferences.getInstance();
await prefs.setInt('global_stars', updatedStars);
```

مشاكل هذا الأسلوب:

- **لا مركزية**: لكل جهاز نسخته الخاصة من البيانات.
- **لا تحليلات**: لا يمكن لوليّ الأمر متابعة الطفل من جهاز آخر.
- **هشاشة**: حذف التطبيق = ضياع كل التقدّم.
- **لا توسّع**: لا يمكن إضافة تسجيل دخول، عدّة أطفال، تقارير، لوحة معلّم… إلخ.

الحل: نقل «مصدر الحقيقة» (Source of Truth) إلى قاعدة بيانات على سيرفر، ويبقى
`SharedPreferences` مفيداً فقط لـ:
- تخزين **إعدادات الاتصال** (IP/البورت).
- التخزين المؤقّت (cache) للعمل دون اتصال.

## 2. المعمارية المستهدفة (3 طبقات)

```
┌─────────────────────────────────────────────────────────────┐
│  الطبقة 1: العميل — Flutter (untitled1/)                      │
│  ─────────────────────────────────────────────────────────   │
│  Widgets / Screens                                           │
│      │                                                       │
│      ▼                                                       │
│  Repositories  (ChildRepository, ProgressRepository …)       │
│      │            يستبدلان ScoreManager/ProgressManager       │
│      ▼                                                       │
│  ApiClient (package:http)  ── يقرأ baseUrl من ApiConfig       │
│      │                                                       │
│      ▼  HTTP (JSON)                                          │
└──────┼──────────────────────────────────────────────────────┘
       │   http://<IP>:80/edufun/api/...
┌──────▼──────────────────────────────────────────────────────┐
│  الطبقة 2: الخادم — PHP على XAMPP (htdocs/edufun/)            │
│  ─────────────────────────────────────────────────────────   │
│  api/  ← نقاط النهاية (Endpoints) + CRUD + التحقق + CORS      │
│      │                                                       │
│      ▼  PDO                                                  │
└──────┼──────────────────────────────────────────────────────┘
       │   SQL
┌──────▼──────────────────────────────────────────────────────┐
│  الطبقة 3: البيانات — MySQL/MariaDB (XAMPP)                   │
│  قاعدة: edufun_db  (children, games, progress, scores …)     │
└─────────────────────────────────────────────────────────────┘
```

كل طبقة لا تعرف تفاصيل الطبقة التي بعدها إلا عبر **عقد واضح**:
- بين فلاتر والـ PHP العقد هو **REST API** (موثّق في الملف 04).
- بين الـ PHP والـ MySQL العقد هو **مخطّط الجداول** (موثّق في الملف 02).

## 3. تدفّق طلب نموذجي (مثال: الطفل أنهى لعبة)

```
الطفل يفوز بـ word_game
        │
        ▼
ProgressRepository.markCompleted(childId, 'word_game')
        │  POST /edufun/api/progress/
        │  body: {"child_id":1,"game_code":"word_game","stars":3}
        ▼
PHP: progress/index.php  →  يتحقق  →  INSERT/UPDATE في جدول game_progress
        │                              →  يزيد children.total_stars
        ▼
استجابة JSON: {"success":true,"data":{...}}
        │
        ▼
فلاتر يحدّث الواجهة (نجوم + فتح المستوى التالي)
```

نفس المنطق الموجود حالياً في `ProgressManager.markGameCompleted()` لكنه ينفّذ على
السيرفر بدل الجهاز.

## 4. مبدأ «اللمسة الخفيفة» في التحويل

لا نعيد كتابة التطبيق. نتبع هذه القاعدة:

1. **لا نلمس الواجهات (Screens)** قدر الإمكان.
2. نُبقي على واجهة `ScoreManager` / `ProgressManager` نفسها (نفس أسماء الدوال)
   لكن نغيّر جسمها الداخلي ليتصل بالـ API بدل `SharedPreferences`.
3. نضيف طبقة `Repositories` + `ApiClient` جديدة.
4. نضيف **وضع بديل (fallback)**: عند فشل الاتصال نقرأ من الكاش المحلي،
   فيبقى التطبيق يعمل حتى لو كان السيرفر متوقفاً.

بهذا يكون التحويل تدريجياً وآمناً. التفاصيل في الملف 05.

## 5. لماذا XAMPP و PHP تحديداً؟

- **XAMPP** يجمع Apache (سيرفر ويب) + MySQL/MariaDB + PHP في حزمة واحدة، مناسب
  للتطوير المحلي على اللابتوب دون إعداد معقّد.
- **المسار `/edufun`**: نضع مشروع الـ PHP داخل `xampp/htdocs/edufun/`، فيصبح
  متاحاً تلقائياً على `http://localhost:80/edufun`.
- **البورت 80** هو بورت Apache الافتراضي؛ لذلك يكفي كتابة الـ IP دون بورت في المتصفح،
  لكننا سنكتبه صراحةً في فلاتر للوضوح.

إعداد XAMPP والشبكة بالتفصيل في الملف 07.
